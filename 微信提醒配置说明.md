# 微信提醒配置说明

## 微信里打开页面

临时测试时，电脑和手机在同一个 Wi-Fi 下，可使用终端显示的局域网地址：

`http://电脑局域网IP:8768/wechat`

把这个地址发到微信，手机点开即可查看招标线索。

长期使用建议部署到云服务器或托管平台，使用公网地址：

`https://你的公网地址/wechat`

这样电脑不开机也能在微信里查看。

## 微信提醒通道

监控脚本：`tender_monitor.py`

支持三种方式，任选一种配置环境变量：

### PushPlus

适合个人微信提醒，推荐先用这个。

1. 打开 PushPlus 官网，微信扫码登录。
2. 复制你的 Token。
3. 在项目里复制 `.env.pushplus.example` 为 `.env`。
4. 填入：

```bash
PUSHPLUS_TOKEN=你的PushPlusToken
TENDER_PAGE_URL=http://127.0.0.1:8768/wechat
```

部署到公网后，把 `TENDER_PAGE_URL` 改成：

```bash
TENDER_PAGE_URL=https://你的公网地址/wechat
```

手动测试：

```bash
python3 tender_monitor.py
```

### 企业微信群机器人

在企业微信群里添加群机器人，复制 Webhook 地址，然后设置：

`WECOM_WEBHOOK_URL=你的企业微信群机器人Webhook`

推荐团队使用这个方式。操作路径通常是：

1. 企业微信群右上角群设置
2. 群机器人
3. 添加机器人
4. 复制 Webhook 地址
5. 在项目里复制 `.env.example` 为 `.env`，填入：

```bash
WECOM_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key
TENDER_PAGE_URL=https://你的公网地址/wechat
```

注意：Webhook 相当于这个群机器人的发送钥匙，不建议发到公开群或公开仓库。

### Server酱

设置：

`SERVER_CHAN_SENDKEY=你的SendKey`

## 手动检查一次

```bash
python3 tender_monitor.py
```

首次运行会把当前线索都视为新增并提醒；之后只提醒新增线索。

## 定时检查

可以用系统定时任务每天运行 `python3 tender_monitor.py`。如果需要我在 Codex 里创建每日检查任务，我可以继续配置。

如果部署在云服务器上，定时任务也应放在云服务器上运行，这样电脑不开机也能提醒。
